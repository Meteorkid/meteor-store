# Meteor Store 最新产品媒体重新采集设计

> 日期：2026-07-05

## 目标

以本机各项目当前代码为唯一版本基准，重新运行并采集 Meteor Store 中全部九个产品的展示素材。已有截图和 GIF 均不直接复用，只有本轮从当前版本实际运行结果中生成的素材可以作为正式媒体。

## 产品范围

1. OmniCrawl
2. Ex-Memory
3. Skeleton Anatomy
4. UI Design System
5. Statux
6. XIsland
7. Tollow
8. XNook
9. Chakra Visualizer

## 协作分工

Claude Code 是主要执行者，负责：

- 识别每个项目的启动方式和运行依赖。
- 启动本机项目并进入具有代表性的真实功能界面。
- 采集、裁切、压缩并归档截图。
- 为适合动态演示的产品录制 GIF。
- 更新 Meteor Store 产品媒体数据和引用路径。
- 记录无法启动、无法进入功能界面或无法录制的原因。

Codex 负责：

- 审核素材是否来自当前真实界面。
- 检查构图、比例、敏感信息、清晰度和文件体积。
- 验收产品列表、详情页及响应式效果。
- 运行测试和生产构建。
- 仅接手 Claude Code 明确无法完成的项目或步骤。

## 每个产品的最低交付

每个产品必须提供：

- `cover.webp`：列表页和详情页主视觉，目标比例 16:10。
- `screenshot-1.webp`：第一张真实功能界面。
- `screenshot-2.webp`：第二张真实功能界面。

当产品包含适合展示的关键交互时，增加：

- `demo.gif`：6–10 秒、可循环、无敏感信息的动态演示。

允许提供第三、第四张截图，但不为凑数量重复相同画面。

## 素材规范

- 所有画面必须来自本机当前代码的实际运行结果。
- 不使用生成式图片伪造软件功能。
- 不展示密钥、Token、私人聊天、真实账户、绝对路径或其他敏感信息。
- 使用统一的 App Store 风格外框和背景包装，但不遮挡核心界面。
- WebP 在可辨识文字的前提下压缩；GIF 优先控制时长、帧率和尺寸。
- 文件统一存放在 `public/products/<product-id>/`。

## 采集流程

每个项目依次执行：

1. 检查本地状态、README、启动脚本和依赖。
2. 启动项目并确认当前版本可用。
3. 选择首页与两个有区分度的核心功能场景。
4. 清理或替换敏感数据。
5. 采集封面和功能截图；必要时录制 GIF。
6. 转为统一格式并检查文件体积。
7. 接入产品数据并在 Meteor Store 中预览。
8. 记录完成状态或阻塞原因。

不得为了截图修改子项目业务行为。若项目无法运行，先记录可复现原因，再由 Codex 判断是否进行最小修复。

## 验收标准

- 九个产品均有本轮重新采集的封面和至少两张真实截图。
- 动态演示仅用于确实能体现交互价值的项目。
- 列表页桌面端、移动端没有图片溢出、变形或明显裁切错误。
- 详情页媒体画廊无缺图，替代文本准确。
- 页面没有泄露敏感信息。
- `npm run lint`、`npm test`、`npm run build` 全部通过。
- 浏览器实际检查产品列表以及九个产品详情页。

## 异常处理

- 启动失败：保留命令、错误摘要和环境要求。
- 需要账户或外部服务：优先使用本地演示模式或安全测试数据，不擅自登录真实账户。
- macOS 原生应用无法自动截图：Claude Code记录阻塞，交由 Codex 使用本机 UI 工具处理。
- HTTPS、自签名证书或摄像头权限阻塞：不绕过系统安全设置，交由 Codex选择安全采集方案。
- 项目描述与实际功能不一致：以当前运行结果为准，单独列出文案修正建议。
