# Meteor Store 首页艺术化改版 · 设计文档

日期：2026-07-07
状态：待用户审阅

## 1. 背景与目标

Meteor Store（Next.js 16 + TypeScript + Tailwind 4）现有首页视觉合格但平淡。目标：

1. **一眼吸引**：Hero 用数学艺术 × 开发者文化的独特视效建立高级感
2. **处处巧思**：全站散布彩蛋，体现店主（大学生，靠代码赚学费）的幽默人格
3. **作者小序**：在特别的位置放一封深情又活泼的店主自述信
4. **人文关怀**：对少数群体（无障碍）和深夜 emo 访客的真诚关照
5. **硬约束**：零新增运行时依赖；不损害性能与无障碍；ICP 合规语境下表达包容

## 2. 设计原则

- **每一屏只有一个视觉主角**，效果分布全站而非堆砌首屏
- 幽默要"藏"：主动找的人获得奖励，不打扰不找的人
- 所有动效可退化：`prefers-reduced-motion` 一票否决
- 关怀不是文案是工程：aria/对比度/键盘可达性必须真实达标

## 3. 首页结构（自上而下）

| 区块 | 视觉主角 | 说明 |
|---|---|---|
| Hero | **ASCII 莫比乌斯环** + 流星雨背景 | 见 §4 |
| Stats/产品/特性 | 数字滚动 + 卡片 3D 倾斜 | 见 §7 |
| **店主的终端**（新增区块） | 可交互假终端 | 见 §5，作者小序入口 |
| CTA | 无限隧道背景 | 见 §7 |
| Footer | 版权行眨眼彩蛋 | 见 §8-⑤ |

## 4. Hero 区

### 4.1 ASCII 莫比乌斯环（视觉主角）
- Canvas 2D 实时渲染：莫比乌斯环参数方程 → 3D 旋转 → 透视投影 → 亮度映射到字符集 `.,-~:;=!*#$@` → 紫粉 HSL 渐变着色
- 位于 Hero 标题上方/背后，缓慢自转；鼠标横移轻微改变转速与视角（±20%）
- 移动端：降字符网格分辨率约 40%；帧率上限 30fps
- `prefers-reduced-motion`：静态单帧字符画

### 4.2 流星雨背景（替换现有 ParticleBackground）
- 三层：静态闪烁星空（~60 颗）、随机流星（每 0.4–1.6s 一颗，紫粉尾迹渐变）、鼠标尾迹（Hero 区内移动拖出细流星轨迹）
- 鼠标静止 10s → 一颗大流星划过 + 「许个愿？」可点击（彩蛋⑧）
- 保留现有 IntersectionObserver 离屏暂停模式
- 现有网格底纹/光晕/文案结构保留

## 5. 店主的终端（作者小序的特别位置）

首页新增区块 `TerminalSection`，样式为 macOS 风格终端窗口（红黄绿圆点、等宽字体、光标闪烁）。真实可输入，命令集：

| 命令 | 回应 |
|---|---|
| `help` | 列出命令，末行「…以及一些没写在这里的」 |
| `story` | 打字机输出两行预告 → 跳转 `/story` |
| `whoami` | 「一位有品位的访客（能找到这个终端的都是）」 |
| `author` | 「白天上课，晚上写代码，梦想是自力更生赚出学费的大学生」 |
| `ls` | `products/  secret.txt` |
| `cat secret.txt` | 「你居然真的 cat 了。奖励你一个秘密：↑↑↓↓←→←→BA」 |
| `sudo give-me-discount` | 「权限不足 :) 但心意收到了。学生党可以邮件我，真的会给折扣。」 |
| `sudo rm -rf /` | 「⚠ 别！这可是我的学费啊！！（进程已被店主的求生欲终止）」 |
| `vim` | 「你进入了 vim。开玩笑的，谁都别想让我实现 :wq」 |
| `coffee` / `奶茶` | 「☕ 本店由奶茶驱动。今日已消耗：2 杯（数据真实）」 |
| `ping` | 「pong! 延迟 0ms，毕竟我就在你浏览器里」 |
| `exit` | 「退出失败：这是网页。你可以关掉标签页（但我会想你的）」 |
| `42` | 「宇宙、生命以及一切的终极答案。你懂的。」 |
| `hug` | 「(っ´▽`)っ 抱一个。今天辛苦了。」 |
| `晚安` | 「晚安。梦里没有 bug，也没有 deadline。」 |
| `emo` | 「emo 是正常的，说明你在认真地活着。不过如果真的很难受，别一个人扛着——找个人说说，或者在反馈树洞里说给我听。」 |
| `clear` | 清屏 |
| 未知命令 | 「command not found: xxx。不过 help 一下也许有惊喜？」 |

无鼠标交互时自动演示模式（循环打字 `help`→`story` 预览），点击后进入真实交互。移动端弹出虚拟命令按钮组（点按代替键入）。

## 6. /story 作者小序页

新路由 `src/app/story/page.tsx`。信纸质感排版（窄栏、衬线感、暗底暖字），逐段打字机/淡入展开。Footer 增加入口「本店店主的一封信」。

> **一封来自店主的信**
>
> 你好呀，陌生人。
>
> 先自我介绍：我是这家店的店主，一名正在读书的大学生。白天在教室里假装听懂高数，晚上在宿舍里真的写代码。
>
> 这家店里的每一件"商品"——爬虫框架、AI 记忆系统、3D 解剖图谱——都是我一行一行敲出来的。没有团队，没有投资人，只有我、一台电脑，和数不清的奶茶空杯。

> 你可能会问：一个大学生，搞这些干什么？
>
> 答案很俗：赚学费。
>

>
> 说实话，每卖出一单，我都会开心很久——不只是因为钱，更因为那意味着：我写的东西，真的帮到了某个陌生人。这种感觉比考试及格还好。（考试及格的快乐只持续五分钟，这个能持续一星期。）
>
> 如果你买了我的工具：谢谢你，真心的。你添的不只是一块学费的砖，还有一个学生"我可以靠自己"的底气。

> 最后——
>
> 无论你是谁、从哪里来、爱着谁、正在经历什么——你在这里都是受欢迎的。

>
> —— 店主 Meteor

## 7. 其他模块升级

- **CTA 区**：无限隧道 Canvas 背景（同心几何框向观者涌来），标题改「进入工具矩阵」
- **404 页**（`not-found.tsx`）：万花筒镜面动画背景 + 「404 · 镜中无此页」/「这个页面可能去打工赚学费了，替它向你道个歉。」按钮「回首页」「看看产品（顺便帮它凑凑学费）」+ 小字「盯着万花筒看 10 秒，有助于原谅一切 404」
- **产品卡片**：鼠标悬停 3D 磁性倾斜（CSS transform，指针位置驱动）
- **Stats 数字**：进入视口时 count-up 滚动动画

## 8. 彩蛋全集（12 项）

① **控制台留言**：页面加载即 `console.log`（样式化），含 `meteor.story()`（跳转 /story）、`meteor.hire()`（打印邮箱+「简历随时奉上，价格公道，童叟无欺」）、`meteor.secret()`（触发流星雨爆发）
② **Konami 秘技**：↑↑↓↓←→←→BA（移动端连点 Logo 7 次）→ 全屏流星雨爆发 6s + toast「☄ 秘技达成：流星雨爆发！这个秘技和你一样，是老派的浪漫」；同会话第 3 次起换「你已经是流星雨管理员了」
③ **终端命令集**：见 §5
④ **404 万花筒**：见 §7
⑤ **Footer 版权行**：默认「© 2026 Meteor Store · 某个还在攒学费的大学生」；hover →「© 2026 Meteor Store · (｡•̀ᴗ-)✧ 被你发现了」；连点 5 次 → toast「再点也不会掉钱出来啦」
⑥ **Loading 文案轮换**：「正在把流星捞上来…」「正在数店主的奶茶钱…」「正在给比特们排队…」「马上好，代码在赶来的路上」
⑦ **表单彩蛋**：反馈占位符「说点什么吧，夸我也行（尤其欢迎夸我）」；bug 类型时提交按钮「提交 bug（对不起对不起）」；订阅占位符「you@example.com（不会拿去换学费的，放心）」；成功 toast「收到！比收到花呗还款提醒还开心」
⑧ **许愿流星**：见 §4.2；点击 → toast「愿望已记录：祝你和店主都心想事成 ☄」
⑨ **标签页标题**：切走 →「☄ 别走嘛，流星还在等你」；切回闪现「☄ 欢迎回来！」后恢复
⑩ **深夜问候**：并入 §9 树洞模式
⑪ **支付成功页**：流星彩带 + 「感谢你！你刚刚为一个大学生的下学期学费添了一块砖 🧱 这不是营销话术，是真的。—— 店主」
⑫ **滚动到底**：首次触底 → 小流星横穿 Footer + 「你居然滚到底了，送你一颗流星」

## 9. 人文关怀系列（4 项）

⑬ **屏幕阅读器悄悄话**：页首 `sr-only` 问候「你好呀，正在用屏幕阅读器的朋友。这个网站的每个按钮和图片我都认真标注过，希望你逛得顺畅。有任何不方便的地方，页脚有我的邮箱，说了我就改。—— 店主」
⑭ **深夜树洞模式**（本地时间 0:00–5:00）：流星减速 30%、色调偏暖橙；问候语随机不重复（「这么晚还醒着？没关系，流星也醒着。」「深夜的网速和思绪一样，容易飘远。慢慢逛，不着急。」「今天不管发生了什么，都过去了。真的。」）；终端 `hug`/`晚安`/`emo` 命令（见 §5）；反馈表单显示树洞提示「深夜的话可以只说给我听。想说什么都行，我都会看。不用留邮箱。」+ type 增加仅深夜出现的 `深夜树洞` 选项
⑮ **给同路人的两行字**（/story 结尾）：见 §6 末段
⑯ **无障碍底线工程**：`prefers-reduced-motion` 全动效退化为静态 + 控制台留言「检测到你偏好减少动画，已为你切换到安静模式 🌙」；状态信息不孤立依赖颜色；对比度 WCAG AA 校对；键盘全可达（终端/彩蛋/许愿流星）；帧率检测连续跌帧自动降粒子数

## 10. 技术方案

- **零新增运行时依赖**：全部 Canvas 2D / CSS transform / Web API，不引入 three.js
- **组件清单**（均 `'use client'`，动效不阻塞 LCP）：

| 组件 | 职责 |
|---|---|
| `MeteorShower.tsx` | 流星雨背景（含鼠标尾迹、许愿流星、深夜暖色模式、爆发模式 API） |
| `AsciiMobius.tsx` | ASCII 莫比乌斯环渲染 |
| `TerminalSection.tsx` | 可交互终端 + 命令注册表 |
| `InfiniteTunnel.tsx` | CTA 隧道背景 |
| `KaleidoscopeCanvas.tsx` | 404 万花筒 |
| `EasterEggs.tsx` | 全局彩蛋收口（console/Konami/标题/滚动到底/深夜问候/`window.meteor`），挂在 layout |
| `CountUp.tsx` | 数字滚动 |
| `TiltCard.tsx` | 3D 磁性倾斜包装器 |
| `lib/motion.ts` | `useReducedMotion`、帧率检测、深夜判定等共享工具 |

- **改造**：`HeroSection`、`CTASection`、`Footer`、`not-found.tsx`、`success` 页、`StatsSection`、产品卡片、feedback/newsletter 表单文案、`layout.tsx`（挂 EasterEggs + sr-only 问候）
- **新增**：`/story` 页；`feedbacks.type` 增加 `night-whisper` 枚举值（Zod schema 同步，数据库 text 字段无需迁移）
- **删除**：`ParticleBackground.tsx`（被 MeteorShower 替代）

## 11. 实现顺序

1. **基础设施**：`lib/motion.ts` + `EasterEggs.tsx` 骨架 + layout 挂载
2. **Hero**:`MeteorShower` + `AsciiMobius` + HeroSection 接线，删旧粒子
3. **终端与小序**：`TerminalSection` + `/story` 页 + Footer 入口
4. **模块升级**：`InfiniteTunnel`/CTA、404 万花筒、`TiltCard`、`CountUp`
5. **彩蛋与关怀**：彩蛋②⑤⑦⑨⑪⑫、树洞模式、sr-only、表单文案
6. **验收**：reduced-motion 全检、移动端降级、Lighthouse（LCP 不劣化）、无障碍对比度、`npm test` + `npm run build`

## 12. 测试与验证

- 单测：终端命令注册表（输入→输出映射）、深夜判定、Konami 序列匹配（纯逻辑可测）
- 手动验收清单：每个彩蛋逐一触发；VoiceOver 朗读 sr-only 问候；`prefers-reduced-motion` 模拟；移动端 Safari/Chrome 帧率
- 性能红线：LCP 不劣化超过 10%；动效 JS 总增量 < 30KB gzip

## 13. 风险与注意点

- 树洞可能收到沉重内容，`emo` 文案已含「别一个人扛着」引导，点到为止
- 包容表达采用普适语言，不使用具体政治符号（ICP 合规语境）
- `feedbacks.type` 枚举扩展需同步 Zod 校验，避免 400
- 动效组件较多，务必逐一接 IntersectionObserver 暂停，防移动端耗电
