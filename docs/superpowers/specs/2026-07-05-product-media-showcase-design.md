# Meteor Store 产品媒体展示设计

> 日期：2026-07-05  
> 状态：已确认设计，待实施

## 目标

将产品列表页和产品详情页从 Emoji 图标展示升级为以真实软件画面为主的 App Store 风格展示，让访客在进入购买或 GitHub 页面前就能理解产品界面、核心操作和适用场景。

成功标准：

- 九个产品在列表页均有统一比例的真实封面或高质量兜底封面。
- 每个可运行产品拥有一段 6–10 秒循环演示和 2–4 张关键截图。
- 产品详情页提供主演示、截图画廊、功能亮点、平台信息和明确行动入口。
- 桌面端、移动端、键盘操作和低带宽环境均能正常使用。
- 媒体不会造成明显布局跳动或不必要的首屏下载。

## 视觉方向

采用 App Store 风格，以真实运行画面为主体，并用统一包装建立品牌一致性：

- 深色渐变背景。
- macOS 窗口、浏览器窗口或终端窗口外框。
- 统一圆角、阴影、留白和标题位置。
- 每个产品继续使用自己的强调色。
- Emoji 图标仅作为小型识别元素，不再承担主要视觉展示。

## 页面结构

### 产品列表页

每张产品卡片包含：

1. 16:10 产品封面。
2. 产品名称、简介和分类。
3. Web、macOS、CLI 等平台标签。
4. 免费或付费状态标签。
5. “查看详情”入口。

桌面端指针悬停时：

- 若产品有演示视频，封面淡出并静音循环播放。
- 指针离开后停止视频并恢复封面。

移动端：

- 默认只加载静态封面。
- 用户主动点击媒体区域后才加载并播放演示。

### 产品详情页

统一按以下顺序展示：

1. 产品名称、定位、平台标签、GitHub 和 Demo 按钮。
2. 大幅主演示视频，静态封面作为 poster。
3. 2–4 张截图组成的横向画廊。
4. 核心功能卡片。
5. 技术特点与适用场景。
6. 定价、下载或获取入口。
7. 相关产品推荐。

截图可以点击放大；放大层支持关闭按钮、Escape 键和焦点管理。

## 素材标准

每个产品的目标素材：

- 1 张 16:10 横版封面。
- 1 段 6–10 秒循环演示。
- 2–4 张关键功能截图。
- 1 份可选 GIF，用于 README 或社交分享，不作为网页主媒体。

目录结构：

```text
public/products/{product-id}/
├── cover.webp
├── demo.webm
├── demo.mp4
├── demo.gif
├── screenshot-1.webp
├── screenshot-2.webp
└── screenshot-3.webp
```

编码与体积目标：

- 封面和截图使用 WebP。
- 单张封面建议控制在 150–300 KB。
- 演示优先使用 WebM，MP4 作为兼容兜底。
- 单段视频目标控制在 2–4 MB。
- 所有媒体保留统一宽高比，避免布局跳动。

## 素材采集计划

### 复用并重新包装

- XIsland：复用 `Assets/demo.gif` 和现有截图。
- XNook：复用 `Assets/demo.gif` 和现有截图。
- Skeleton Anatomy：复用 `docs/screenshots/`，补录 3D 旋转和骨骼选择。
- Statux：复用终端截图，补录状态栏与 TUI 配置操作。

### 本地运行并采集

- OmniCrawl：录制命令行运行、抓取结果和日志输出。
- Ex-Memory：录制 Web/Gradio 界面中的数据导入和对话流程。
- Tollow：录制文件导入、分析和报告输出。
- Chakra Visualizer：录制摄像头授权后的手势和特效触发。

### 暂时使用兜底封面

- UI Design System：本机尚未定位到对应仓库，第一版使用统一设计的营销封面。定位到实际项目后替换为真实画面。

## 数据模型

扩展 `Product`：

```ts
interface ProductMedia {
  cover: string;
  demoVideo?: {
    webm?: string;
    mp4?: string;
    gif?: string;
  };
  screenshots: Array<{
    src: string;
    title: string;
    description?: string;
  }>;
}

interface Product {
  // 现有字段
  media: ProductMedia;
  highlights?: string[];
  platforms?: Array<'Web' | 'macOS' | 'Windows' | 'Linux' | 'CLI'>;
}
```

`cover` 必填。没有真实素材的产品使用本地兜底封面，页面组件不再自行退回 Emoji。

## 组件边界

### `ProductMediaCard`

- 用于产品列表卡片。
- 管理封面和悬停播放。
- 只接收媒体数据，不读取完整产品对象。
- 在触摸设备和减少动态效果模式下不自动播放。

### `ProductHeroMedia`

- 用于详情页主演示。
- 渲染 poster、WebM 和 MP4 source。
- 视频加载或播放失败时保持封面，不显示破损控件。

### `ProductGallery`

- 展示截图列表。
- 管理当前选中截图和放大预览。
- 支持键盘左右切换、Escape 关闭和焦点恢复。

### `ProductHighlights`

- 展示平台标签、适用场景和技术亮点。
- 只负责内容呈现，不处理媒体播放。

## 加载与错误处理

- 列表页只预加载封面，不预加载所有视频。
- 媒体进入视口或发生用户交互后再设置视频 source。
- 视频播放 Promise 被拒绝时静默保留封面。
- 截图缺失时跳过对应项，不显示空白占位。
- 整个产品缺少真实媒体时使用本地兜底封面。
- 尊重 `prefers-reduced-motion` 和浏览器数据节省设置。

## 可访问性

- 所有封面和截图提供准确的中文替代文本。
- 视频不自动播放声音，不依赖声音传达信息。
- 截图放大层提供可见关闭按钮和正确的 dialog 语义。
- 所有媒体交互支持键盘。
- 焦点样式清晰，关闭预览后恢复到原截图。

## 验证方案

自动检查：

- ESLint。
- TypeScript。
- 现有 Vitest 测试。
- Next.js 生产构建。
- 媒体数据完整性测试：九个产品均有封面，所有本地路径均存在。

浏览器检查：

- 产品列表页桌面悬停播放。
- 移动端按需播放。
- 产品详情主演示加载和失败兜底。
- 截图画廊点击、键盘切换和关闭。
- 1440px、768px、390px 三种视口。
- `prefers-reduced-motion` 模式。
- 网络限速下首屏仍优先显示封面。
